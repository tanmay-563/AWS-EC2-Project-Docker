Fundamentals of Information Retrieval
Search · Index · Rank · Evaluate
Information Retrieval (IR) is the science of obtaining relevant information from large unstructured collections—particularly text. At its core, IR answers: "Given a user query, which documents satisfy the information need?" IR forms the backbone of search engines, QA systems, and modern NLP pipelines.

01
Preprocessing
Text must be cleaned and normalized before indexing. Key steps:
  • Tokenization — split text into tokens
  • Stop-word removal — filter noise words
  • Stemming / Lemmatization
  • Case folding & normalization

02
Inverted Index
Core data structure of IR. Maps each term → list of documents (postings list).
  • Dictionary: vocabulary of all terms
  • Postings: docIDs + positions
  • Boolean queries via set intersection
  • Supports phrase & proximity search

03
Ranking Models
Determines document relevance score for a query:
  • Boolean — exact match (no ranking)
  • Vector Space Model (VSM)
  • Probabilistic — BM25
  • Neural — BERT, DPR, ColBERT
TF-IDFBM25Dense
Model 01
TF-IDF
Term Frequency × Inverse Document Frequency. Rewards terms frequent in a doc but rare across the corpus.
tf(t,d) × log(N / df(t))
Model 02
BM25 (Okapi)
Probabilistic model with document length normalization. Industry standard for sparse retrieval.
IDF × (tf·(k+1)) / (tf + k·(1-b+b·|d|/avgdl))
Model 03
Vector Space
Represents docs & queries as vectors. Cosine similarity measures the angle between them.
sim(q,d) = (q·d) / (|q|·|d|)

Key Trade-off: Precision & Recall are inversely related. Raising the retrieval threshold → higher precision, lower recall. The P–R curve captures this. F1 balances both.
